**StatsSection.tsx — Fix counter animation (BUG-01):**
The `useCountUp` hook uses `Math.floor(easeOut * end)` which can land on `end - 1` due to floating-point precision when `easeOut` is very close to but not exactly 1. Fix by ensuring the final frame always sets the count to exactly `end`:

In the `useCountUp` function, after the RAF loop completes (when `progress >= 1`), explicitly call `setCount(end)` instead of relying on `Math.floor(easeOut * end)`. Change the condition so that when `progress >= 1`, it sets `setCount(end)` directly and does NOT call `requestAnimationFrame` again. This guarantees "13+" displays, not "12+".
